# 在线答题与智能测评平台 PRD 产品需求文档

版本：v1.0  
适用范围：在线答题网站 / 题库系统 / 考试系统 / Hydro 编程题测评接入 / AI 学情分析预留

---

## 一、项目背景

本项目面向少儿编程、信息科技、算法基础、Python、C++、Scratch、Arduino 等课程场景，建设一个在线答题与智能测评平台。

平台需要支持普通题型在线答题，同时为后期编程题测评预留 Hydro 接入能力，并为 AI 学情分析、薄弱点总结、针对性练习推荐预留数据结构和接口。

系统第一阶段目标是完成：

1. 题库管理。
2. 题目分类。
3. 随机组卷。
4. 在线答题。
5. 自动判分。
6. 成绩统计。
7. 错题本。
8. 数据导出。
9. 试卷导出。
10. Hydro 和 AI 能力预留。

在线答题与智能测评平台。

## 三、核心目标

### 3.1 第一阶段目标

完成基础答题系统闭环：

```txt
题库 → 组卷 → 发布考试 → 学生答题 → 自动判分 → 成绩统计 → 错题本 → 数据导出
```

### 3.2 第二阶段目标

增强考试和批改能力：

1. 主观题人工批改。
2. 文件上传题。
3. 每人随机卷。
4. 选项随机。
5. 试卷 PDF / Word 导出。
6. 班级统计分析。
7. 题目正确率分析。

### 3.3 第三阶段目标

接入 Hydro 编程题测评：

1. 编程题题型。
2. Hydro 题目绑定。
3. 代码提交。
4. 判题结果同步。
5. 编程题分数汇总。

### 3.4 第四阶段目标

接入 AI 学情分析：

1. 学生个人报告。
2. 班级整体报告。
3. 薄弱知识点分析。
4. 推荐练习生成。
5. 教师讲评建议。

## 五、功能需求

### 5.1 登录认证

功能说明：

1. 用户通过账号密码登录。
2. 系统区分管理员、教师、学生等角色。
3. 登录后根据角色展示不同菜单。
4. Token 过期后支持刷新。
5. 用户退出后 Token 失效。

验收标准：

1. 正确账号密码可以登录。
2. 错误账号密码不能登录。
3. 禁用用户不能登录。
4. 不同角色登录后菜单不同。
5. 未登录不能访问受保护页面。

---

### 5.2 课程管理

功能说明：

课程用于组织题库、知识点、考试和统计。

字段包括：

1. 课程名称。
2. 课程编码。
3. 课程描述。
4. 封面图。
5. 状态。
6. 排序。

示例课程：

1. Scratch。
2. Python。
3. C++。
4. Arduino。
5. 信息科技。
6. 算法基础。

验收标准：

1. 管理员可以创建课程。
2. 管理员可以编辑课程。
3. 管理员可以禁用课程。
4. 禁用课程不能继续新增考试。
5. 题目必须绑定课程。

---

### 5.3 知识点管理

功能说明：

知识点采用树结构。

示例：

```txt
Python 基础
  ├─ 输入输出
  ├─ 变量
  ├─ 条件判断
  ├─ 循环
  ├─ 列表
  └─ 函数
```

功能包括：

1. 新增知识点。
2. 编辑知识点。
3. 删除知识点。
4. 调整排序。
5. 绑定课程。
6. 支持父子级。

验收标准：

1. 知识点必须属于某个课程。
2. 题目可以绑定一个或多个知识点。
3. 组卷时可以按知识点筛选题目。
4. 统计时可以按知识点计算正确率。

---

### 5.4 标签管理

功能说明：

标签用于灵活分类题目。

示例标签：

1. 易错题。
2. 竞赛题。
3. 入门题。
4. 拓展题。
5. 课堂练习。
6. 期末复习。
7. 蓝桥杯。
8. WRC。

验收标准：

1. 管理员可以创建标签。
2. 题目可以绑定多个标签。
3. 组卷时可以按标签筛选。
4. 题库列表可以按标签查询。

### 5.5 题库管理

#### 5.5.1 支持题型

第一阶段支持：

1. 单选题。
2. 多选题。
3. 判断题。
4. 填空题。
5. 简答题。

第二阶段支持：

1. 编程题。
2. 材料题。
3. 文件上传题。
4. Scratch 项目题。
5. Arduino 项目题。

#### 5.5.2 题目字段

题目应包含：

1. 题干。
2. 题型。
3. 选项。
4. 正确答案。
5. 解析。
6. 默认分值。
7. 难度。
8. 课程。
9. 知识点。
10. 标签。
11. 状态。
12. 创建人。
13. 审核人。
14. 版本号。
15. 是否允许选项随机。

#### 5.5.3 题目状态

```txt
draft             草稿
pending_review    待审核
published         已发布
disabled          已停用
archived          已归档
```

#### 5.5.4 题目版本

需求说明：

1. 题目修改后必须生成新版本。
2. 已发布考试使用题目快照。
3. 历史考试不受题库修改影响。
4. 可查看历史版本。

验收标准：

1. 老师修改题目后，不影响已经发布的考试。
2. 历史答题记录可以还原当时题目。
3. 试卷导出应使用发布时的题目快照。

#### 5.6.1 试卷类型

1. 固定试卷。
2. 规则组卷。
3. 随机试卷。
4. 练习卷。

#### 5.6.2 试卷功能

教师可以：

1. 新建试卷。
2. 编辑试卷。
3. 添加题目。
4. 删除题目。
5. 设置题目分值。
6. 设置考试时长。
7. 设置题目乱序。
8. 设置选项乱序。
9. 预览试卷。
10. 发布试卷。
11. 复制试卷。
12. 归档试卷。
13. 导出试卷。

验收标准：

1. 草稿试卷可以编辑。
2. 已发布试卷不能直接破坏历史快照。
3. 试卷总分自动计算。

### 5.7 规则组卷

功能说明：

教师可按规则自动抽题。

规则条件包括：

1. 题型。
2. 课程。
3. 知识点。
4. 标签。
5. 难度。
6. 题目数量。
7. 每题分值。
8. 是否排除已选题。

示例：

```txt
单选题：知识点为变量和循环，难度 1-2，抽 10 题，每题 2 分。
多选题：知识点为条件判断，难度 2-3，抽 5 题，每题 4 分。
填空题：知识点为输入输出，难度 1-3，抽 5 题，每题 4 分。
```

验收标准：

1. 题库数量不足时不能发布。
2. 系统提示具体哪条规则题量不足。
3. 同一试卷中不能重复抽取同一道题。
4. 规则组卷结果可以预览。
5. 组卷后可以手动微调。

---

### 5.8 随机组卷

功能说明：

随机组卷分两种：

1. 老师发布时随机生成一套固定试卷。
2. 学生进入考试时，每人生成不同试卷。

每人随机卷要求：

1. 学生第一次进入考试时生成个人试卷实例。
2. 刷新页面不能重新抽题。
3. 退出后再次进入仍然读取原实例。
4. 每个学生可以看到不同题目或不同顺序。
5. 历史记录必须能还原学生实际看到的试卷。

验收标准：

1. 同一学生多次进入考试，试卷不变化。
2. 不同学生可按规则生成不同试卷。
3. 题目和选项顺序保存在个人试卷实例中。
4. 判分不受选项展示顺序影响。

### 5.9 在线考试

#### 5.9.1 考试配置

考试字段包括：

1. 考试名称。
2. 绑定试卷。
3. 绑定课程。
4. 绑定班级。
5. 开始时间。
6. 结束时间。
7. 答题时长。
8. 作答次数。
9. 是否允许迟到进入。
10. 是否允许提前交卷。
11. 是否显示成绩。
12. 是否显示答案。
13. 是否显示解析。
14. 是否乱序。
15. 是否记录切屏。
16. 是否自动交卷。

#### 5.9.2 学生答题

学生答题页需要支持：

1. 题目展示。
2. 题号导航。
3. 已答 / 未答状态。
4. 标记疑问题。
5. 倒计时。
6. 自动保存。
7. 手动保存。
8. 提交确认。
9. 刷新恢复。
10. 移动端适配。

验收标准：

1. 学生只能进入自己有权限的考试。
2. 考试未开始不能进入。
3. 考试结束后不能进入。
4. 答题时长结束后自动交卷。
5. 刷新页面不丢失答案。
6. 提交后不能继续修改答案。
7. 重复点击提交不会重复判分。

### 5.10 判分

#### 5.10.1 客观题自动判分

支持：

1. 单选题自动判分。
2. 多选题自动判分。
3. 判断题自动判分。
4. 填空题自动判分。

#### 5.10.2 多选题评分模式

支持：

1. 严格模式：全对得分，否则 0 分。
2. 少选部分分：少选且无错选，按比例给分。
3. 选项独立计分，后期扩展。

第一阶段建议默认使用严格模式，可配置少选部分分。

#### 5.10.3 填空题评分规则

支持：

1. 完全匹配。
2. 忽略大小写。
3. 忽略首尾空格。
4. 多个可接受答案。
5. 数值误差范围。
6. 正则匹配，后期扩展。

#### 5.10.4 主观题人工批改

功能包括：

1. 查看学生答案。
2. 查看参考答案。
3. 输入得分。
4. 输入评语。
5. 保存批改结果。
6. 批量发布成绩。

验收标准：

1. 客观题提交后自动判分。
2. 主观题进入待批改状态。
3. 主观题批改完成后汇总总分。
4. 批改人和批改时间必须记录。
5. 成绩发布前学生不能看到未发布结果。

---

### 5.11 Hydro 编程题接入

第一阶段：

1. 编程题作为题型预留。
2. 支持填写 Hydro 题目链接。
3. 学生可跳转 Hydro。
4. 老师可手动录入结果。

第二阶段：

1. 平台绑定 Hydro 账号。
2. 同步 Hydro 提交结果。
3. 编程题成绩汇总到考试总分。

第三阶段：

1. Hydro 插件回调。
2. 自动同步判题状态。
3. 自动生成编程题分析。

验收标准：

1. 平台可以创建编程题。
2. 编程题可以绑定 Hydro 地址。
3. 学生可以进入 Hydro 测评。
4. 平台可以保存 Hydro 结果摘要。
5. Hydro 同步失败可以重试或人工修正。

2. AI 分析失败不影响考试成绩。
3. AI 报告可以重新生成。

## 十、当前实现状态，2026-06-30 更新

### 10.1 已完成或基本完成

1. 登录认证与 Token 刷新：已支持账号密码登录、刷新登录状态、角色菜单切换和退出。
2. 角色与数据范围：已支持管理员、教师、助教、学生等角色；教师/助教主要考试、班级、导出数据已按班级范围校验。
3. 课程、知识点、标签：已支持增删改查、批量导入、课程-知识点树形组织。
4. 题库管理：已支持单题录入、批量录入、Markdown、代码块、数学公式、化学式、图片/附件、重复检测、状态管理、分页、排序、按标签/知识点筛选。
5. 题库导入导出：CSV/JSON 用于数据迁移，ZIP 可打包题目 Markdown、元数据和附件资源；导入导出字段已尽量保持闭环。
6. 试卷管理：已支持创建、预览、调整题目、手动排序、按标签/知识点/课程条件组卷、试卷导入、试卷文档导出和迁移包导出。
7. 考试管理：已支持创建、编辑、发布、取消发布、批量状态更新、模拟学生、教师试答、成绩/排名查看。
8. 考试公告：已从 JSON 配置迁移为独立公告表，支持版本记录、学生阅读确认、未读筛选、未读名单导出和站内提醒。
9. 学生答题：已实现专注答题页、自动保存、提交次数限制、题目随机、选项随机、考试结果显示策略。
10. 自动判分与主观批改：客观题自动判分已覆盖选择、判断、填空等；主观题/文件类题型进入批改中心。
11. 错题本：已支持来源追踪、掌握状态、练习、错题组卷、错题导出、教师按错题频次组卷，以及按课程/班级/知识点配置复习提醒规则。
