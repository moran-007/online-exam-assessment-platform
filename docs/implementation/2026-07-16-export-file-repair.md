# 题目与试卷导出文件修复实施记录

## 结论

题目和试卷导出后无法打开的问题已修复。问题不在 PDF、Word 或 ZIP 生成器，而在全局统一响应拦截器：它把 `StreamableFile` 再包装为 JSON，同时控制器仍声明原始文档 MIME 类型，导致浏览器把 JSON 内容保存为 `.pdf`、`.docx` 或 `.zip`。

## 修复范围

- 二进制流直接穿过统一响应拦截器，普通 API 仍保持 `{ code, message, data }` 契约。
- 浏览器 Blob 下载统一使用显式挂载、点击、移除的临时链接，兼容更多浏览器和自动化环境。
- 导出集成测试不再只检查“文件大于 20 字节”，而是按格式验证实际内容。
- 增加题库和试卷的浏览器点击回归流程。

## 格式验证规则

| 格式 | 验证方式 |
| --- | --- |
| JSON | 使用 JSON 解析器读取 |
| CSV | 去除 UTF-8 BOM 后验证表头和分隔内容 |
| XLSX | 验证 ZIP 本地文件头和中央目录结束记录 |
| DOCX | 验证 ZIP 本地文件头和中央目录结束记录 |
| ZIP | 验证 ZIP 本地文件头和中央目录结束记录 |
| PDF | 验证 `%PDF-` 文件头和 `%%EOF` 结束标记 |

## 点击验收

Chromium 浏览器完成以下真实操作：

1. 登录管理员账号并进入题库管理。
2. 选择题目，通过“操作 → 下载”生成 JSON 导出任务。
3. 进入导出中心，等待任务成功，点击“下载”，解析返回 JSON。
4. 在试卷导出列表选择学生版 PDF。
5. 等待任务成功，点击“下载”，验证 PDF 文件头和结束标记。

## 维护约束

- 后续新增文件下载接口必须返回顶层 `StreamableFile` 或直接使用原始响应，不得把文件字节塞入普通 JSON DTO。
- 新增导出格式时必须补充内容级验证，不能只检查状态码、扩展名或文件大小。
- 前端统一复用 Blob 下载工具，避免各页面重复实现临时链接逻辑。
