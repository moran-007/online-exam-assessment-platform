# 阶段 3：学生考试阶段总结实施计划

> 日期：2026-07-16  
> 状态：进行中；Gate A 已通过

## 目标与边界

- 基于已评分考试、题型、知识点、错题和编程判题结果生成学生阶段总结。
- 当前阶段不读取考勤、课次、作业、课堂表现、教学备注或家长数据，也不得把缺失数据解释成缺勤或学习态度问题。
- AI 只生成可编辑草稿；人工审核后才能发布，学生只能查看本人已发布内容。
- 首版只支持单个学生生成；批量接口只提供任务数与 Token 预估，不开放全校批量执行。

## 数据范围契约

生成范围由以下字段组成：

- `studentId`：必填；
- `courseId`：可选；
- `from`、`to`：可选时间范围；
- `examIds`：可选，最多 20 场；
- `configId`：可选，不传时按个人默认、系统默认顺序选择；
- `maxTokens`：可选，不传时使用模型配置安全上限。

数据集固定使用每场考试最新一份已评分答卷，包含：

- 成绩与得分率趋势；
- 题型答题数、正确率和得分率；
- 知识点答题数、正确率和得分率；
- 错题次数和掌握状态，不发送错误答案正文；
- 编程提交数、通过数、通过率和得分；
- 明确的 `dataCoverage.includes` 与 `dataCoverage.excludes`；
- 每个具体数值对应的 EvidenceRef。

## 权限规则

- 超级管理员和管理员可访问全部学生。
- 教师和助教只能访问自己任教班级中的学生，或参加了其本人创建的公开考试的学生。
- 即使学生身份可访问，考试仍按教师的班级和本人创建范围再次过滤。
- 显式选择的考试只要有一场越权，整次请求拒绝，不静默缩小范围。
- 学生查询只允许 `subjectId` 等于当前用户且状态为已发布的学生总结。

## API 目标

- `GET /api/v1/ai-summaries/students/:studentId/preview`
- `POST /api/v1/ai-summaries/students`
- `GET /api/v1/students/:studentId/ai-summaries`
- 复用通用详情、编辑、审核、发布、撤回和任务查询接口。
- `GET /api/v1/me/ai-summaries` 同时返回本人可见的考试总结和学生阶段总结。

## 生命周期与成本

- 学生总结使用独立 Prompt、Schema 和数据集版本。
- 幂等身份包含学生、范围、数据哈希、Prompt、Schema、模型、输出上限和生成键。
- 相同输入与生成参数复用成功结果；显式重新生成使用新生成键。
- 默认通过模型配置控制输出，建议配置为 500–800 Token；页面显示请求上限、实际用量、保守预留和本地剩余额度。
- 任何失败重试和批量预估都需要二次确认。
