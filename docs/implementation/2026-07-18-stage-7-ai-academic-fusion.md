# 阶段 7：AI 与教务深度融合实施记录

## 完成结论

阶段 7 已完成。考试总结、学生综合总结、班级总结、家长报告和课堂助手已经共用任务、模板、权限、证据引用、审核发布与质量评测体系；测评与教务指标也已进入统一看板。

核心边界：

- 统计事实由确定性查询产生，AI 只能解释，不能改写分数、出勤或课时。
- 学生综合总结使用 v2 数据集，并保留数据覆盖区间；少数据时降级为事实卡片。
- 班级总结只提供班级聚合，不泄露未经授权的学生明细。
- 家长报告只使用已发布且允许家长查看的内容，必须经过教师审核后发布。
- 课堂助手只生成本地教学记录草稿，应用后仍需教师保存、提交和发布。
- AI 服务不可用时，看板、教学记录、学习门户和历史已发布内容仍可正常使用。

## 数据集与隐私边界

| 数据集 | 主要输入 | 输出约束 |
| --- | --- | --- |
| `StudentSummaryDataset` v2 | 已发布考试、出勤、已完成课次、课堂目标、表现和作业 | 显示覆盖区间；低覆盖率使用事实卡片；每项事实可追踪 `EvidenceRef` |
| `ClassSummaryDataset` | 考试、知识点、出勤、课次和作业的班级聚合 | 不输出学生姓名或无授权个体明细 |
| `ParentReportDataset` | 家长可见的已发布考试、教学记录和 AI 总结 | 排除草稿、内部观察和隐藏成绩；使用中性、不标签化模板 |
| `LessonAssistantDataset` | 当前课次、课堂目标和教师已有教学记录 | 仅生成教学记录措辞、作业和下次计划草稿，不自动保存或发布 |

所有生成类型使用统一 Prompt Factory 和结构化输出校验，输出必须包含 `schemaVersion`。数据集、模板和模型版本随任务记录，v1/v2 可追溯。

## 生命周期、权限与页面

新增总结类型：`class`、`parent_report`、`lesson`。它们复用既有 `DRAFT → IN_REVIEW → APPROVED → PUBLISHED` 生命周期、版本历史、撤回和重新生成机制。课堂助手停留在草稿使用场景，不开放自动发布。

新增细粒度权限：

- `ai.summary.class.generate`
- `ai.summary.parent-report.generate`
- `ai.summary.lesson.generate`
- `ai.feedback.create`
- `ai.quality.read`
- `ai.quality.manage`
- `dashboard:read`

服务端同时执行权限、任课范围、学生本人和家长关联关系校验。学生及家长只能反馈其有权查看的已发布总结，不能借由总结 ID 读取未发布内容。

前端入口包括：

- 管理员/教师首页的测评与教务融合看板及确定性下钻；
- 班级详情中的班级总结和家长报告；
- 教学记录抽屉中的课堂助手及“应用到本地草稿”；
- 学生/家长学习门户中的已发布总结与质量反馈；
- AI 设置页中的调用量、Token、估算成本、缓存、反馈和回归评测。

主要接口：

- `GET|POST /ai-summaries/lessons/...`
- `GET /classes/:classId/ai-summaries`
- `GET /students/:studentId/parent-reports`
- `GET /lesson-sessions/:sessionId/ai-assistant`
- `GET /ai-quality/dashboard`、`GET /ai-quality/feedback`
- `POST /ai-summaries/:id/feedback`

OpenAPI 文档和前端生成客户端已同步。

## Token、成本与模型兼容

本次输出上限为空时，不增加单次请求限制，使用模型配置的安全上限；填写数值时才覆盖本次请求。界面分别展示请求上限、供应商实际用量和供应商未报告时的保守预留，避免把成本参考值误当成必填限制。

供应商返回准确 usage 时按实际输入/输出 Token 记账；未报告时按请求上限保守预留，并明确标记为估算。1000 Token 仅作为成本参考，不是强制默认值。供应商和模型可配置单价，质量看板按实际或估算用量计算成本。

真实 DeepSeek 调用成功：显式输出上限 1000，供应商报告输入 2516、输出 715 Token，并生成可应用的课堂草稿。阿里云配置此前连接测试成功，但总结请求曾出现超时且未报告 usage；系统已正确保守记账和展示错误，不把供应商超时伪装为成功。实施记录不保存或展示任何密钥。

## 质量、缓存、反馈与回归

- 质量看板按功能、模型和模板聚合成功率、调用量、Token 和估算成本。
- 缓存事件区分命中与未命中，可验证重复任务的节省效果。
- 学生和家长可对已发布总结提交评分、结论和事实错误说明；教师或管理员可处置并留痕。
- 模型切换前可运行黄金样本回归，记录通过率、失败原因、模型和模板版本。
- 供应商异常、结构化输出校验失败和超时均记录明确状态，不生成可发布的伪结果。

## 运维与回滚

- AI 功能可按权限或入口独立撤下，不影响确定性教务查询。
- 失败任务保留输入版本、模型、模板和错误信息，可以安全重新生成。
- 反馈、回归和缓存记录为辅助运维证据，不反向修改业务事实。
- 家长可见范围、学生关系或发布规则变化后，必须重新执行学生、关联家长和未关联家长三类越权回归。
