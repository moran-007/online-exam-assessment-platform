# 测评系统部署与运维参考

## 定位与边界

测评能力与教务能力共用账号、课程、班级、权限、文件、通知、AI 配置和审计。当前可靠部署单元是同一个 NestJS 后端、Vue 前端和 PostgreSQL 数据库；考试课程引用共享课程主键，不维护第二套课程目录。

测评侧主要页面：

- `/questions`、`/question-import`、`/question-bank`
- `/papers`、`/exams`、`/grading`
- `/exports`、`/statistics`、学生考试与错题本
- AI 考试/学生/班级总结和模型质量管理

主要 API 领域包括课程知识点、标签、题目、试卷、考试、答题、自动与人工评分、重判、统计、导出、外部评测和 AI 总结。

## 部署模式

### 统一部署（推荐）

本地通过 `scripts\start-platform.ps1 -NoBrowser` 启动；固定使用后端 3000、前端开发入口 5173。生产环境按 `deploy/ONE_CLICK_DEPLOY.md` 配置 PM2、Nginx、PostgreSQL、备份定时器和健康检查。

### 测评入口按需开放

需要独立测评门户时，可为同一前端产物配置单独域名或路径，并给账号只授予题目、试卷、考试、批改、统计和导出权限。该做法分离入口和岗位，不拆分共享课程与班级数据，也不改变内部端口。

受限入口应使用管理员、教师或助教角色；超级管理员绕过普通权限过滤，不应用于受限运营账号。

### 物理拆分限制

当前版本不支持测评服务和教务服务分别写入独立数据库。未来若拆分，课程、班级成员、考试报名、成绩可见性、附件和 AI 数据集都必须有版本化契约、幂等事件和可回放对账；在这些条件满足前，共享数据库的单写模型更安全。

## 发布配置重点

- 生产密钥只从部署 Secret 注入；已暴露供应商 Key 必须在供应商控制台撤销后重建。
- AI 调用级 Token 留空表示不增加额外上限；仍受模型能力和所选配置上限约束。
- `/uploads` 不作为静态目录公开；题目素材、导出和教学附件都通过鉴权接口输出。
- 外部 Judge、Scratch Runtime 和对象存储均为可选适配器；未启用时本地核心流程仍须可用。
- Swagger 生产环境按需关闭；需要时只向可信管理网络开放。

## 最小权限集合

- 命题人员：课程/知识点只读、题目读取与创建、素材上传。
- 组卷人员：题目只读、试卷管理和规则校验。
- 考务人员：考试管理、报名与发布，不自动获得批改权限。
- 批改教师：待批记录读取、评分写入和必要的数据范围。
- 数据人员：统计读取和导出任务权限；不能读取未授权学生。
- 学生：公开题库、本人考试、本人结果和错题本。

## 发布前验收

1. 课程、知识点、题目和材料子题可创建、修改、查看和按依赖顺序删除。
2. 试卷组题、发布、考试进入、自动保存、提交和超时结束均可用。
3. 客观题、主观题、评分规则和重判保留版本与历史。
4. 题目、试卷和成绩导出实际下载后能被目标软件打开，不能只检查 HTTP 200。
5. AI 总结可生成草稿、人工修改、审核、发布和撤回；Schema 与 Token 用量可审计。
6. 学生、教师、管理员数据范围和按钮权限与后端拒绝结果一致。
7. 运行完整质量门禁，并验证备份和隔离恢复。

## 备份、回滚与故障处理

考试提交、答案、评分、导出、AI 用量和教务课时可能互有关联，必须整库备份。发布回滚不能删除已提交答卷或覆盖评分历史；Schema 回退优先使用向前修复版本。

具体日志、导出文件、AI 超时、Token 和权限排查见 `troubleshooting-and-maintenance.md`。接口契约以 `docs/openapi.json` 和运行时 `/api/docs` 为准。
