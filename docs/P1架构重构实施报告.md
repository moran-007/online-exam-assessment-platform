# P1 架构重构实施报告

> 实施日期：2026-07-15
> 实施分支：`codex/p1-architecture-refactor`

## 1. 结果摘要

本轮将系统从巨型 Service/页面和弱契约调用演进为类型化模块单体与领域化前端。既有 NestJS、Prisma、PostgreSQL、Vue 3 和 Vite 技术栈、数据库历史 migration、URL、角色权限及业务数据保持兼容；迁移完成的旧巨型实现已删除，没有保留双轨 Facade 或旧业务 `api()` 调用。

## 2. 主要交付

| 领域 | 实施结果 |
| --- | --- |
| 后端用例 | 题库、学生考试、考试、Hydro、导出、试卷、用户、阅卷已拆为查询/命令/基础设施用例 |
| 前端 | 题库导入、题库、考试作答、考试管理、统计、导出、Hydro、试卷和 AI 已进入 feature 分层 |
| 契约 | Swagger CLI plugin、显式响应 DTO、版本化 OpenAPI、Orval Fetch 客户端和差异门禁 |

## 3. 关键设计边界

- Controller 直接依赖用例类，写用例持有事务，不增加聚合 Facade。
- 跨领域只通过窄接口协作，不引入通用 Repository。
- 前端路由 View 是编排壳；业务状态进入 feature composable，组件不拼接业务 URL。
- PostgreSQL 队列和本地对象存储保留单实例默认实现；多实例触发条件出现后可替换为 BullMQ/Redis 和 S3/MinIO。
- Hydro、Scratch 等专业运行时通过 Gateway/任务适配，不强行并入 NestJS 进程。

## 4. 质量门禁

CI 顺序覆盖：Prisma 空库迁移和 drift、Lint、架构边界、漏洞/许可证、OpenAPI 差异、`vue-tsc`、前后端构建、包体积、关键代码覆盖率、Jest 和 Playwright。

关键单元覆盖门禁当前要求：新增/关键基础设施全局至少 80% statements/lines、70% branches/functions；指标状态逻辑至少 90% branches。全仓覆盖率不以虚假的一次性高数字代替，后续按领域逐步扩大收集范围。

最终严格验收补充了 Hydro 全生命周期隔离测试：加密凭据登录、机器人阻断、公开题目拉取、代码提交、待判结果轮询、最终成绩事务回写均使用模拟 HTTP/Prisma 验证，不访问外部站点。当前单元测试共 56 项，关键改动代码覆盖率为 92.98% lines、77.71% branches。

## 5. 数据与回滚

- 历史 Prisma migration 和生产数据不删除。
- Hydro 旧明文列在所有记录成功验证后通过带前置条件的 migration 删除。
- 导出任务新增租约字段，不改变既有任务业务含义。
- AI 配置是新增表，不影响原有业务；回滚应用版本前应先备份新增配置，再按 migration 策略处理。
- 备份分支 `codex/backup-pre-p1-architecture-20260715` 保留重构前完整快照。

## 6. 尚未越界实施的事项

- 未引入微服务、Nx/Turborepo、全局状态库或重建数据库。
- 未因“以后可能多实例”提前引入 Redis、BullMQ、MinIO/S3。
- 教务系统本轮完成可行性、领域模型、迁移和切换方案；实际排课、考勤、课消和家长门户按独立计划实施。
- AI 真实连接只在管理员安全录入有效 Key 后执行，不把 Key 写入代码、文档或自动化测试。
