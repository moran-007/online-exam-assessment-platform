# 运行可观测性与告警手册

## 1. 入口与追踪

- 存活探针：`GET /api/v1/health/live`。
- 就绪探针：`GET /api/v1/health/ready`，包含数据库连通性检查。
- Prometheus 指标：`GET /api/v1/metrics`。
- HTTP 请求接受合法的 `X-Request-ID`，服务端会在响应中回传；未提供时自动生成 UUID。
- 统一错误响应包含 `requestId`。导出后台任务使用 `export:<taskId>` 作为 correlation id。
- 日志必须保持结构化；禁止记录 Token、Cookie、密码、API Key、Hydro 凭据、学生完整答案和第三方完整响应。

## 2. 核心指标

| 指标 | 说明 |
| --- | --- |
| `online_exam_http_requests_total` | 按方法、路由和状态码统计请求数 |
| `online_exam_http_request_duration_seconds` | HTTP 延迟直方图 |
| `online_exam_prisma_queries_total` | Prisma 查询数 |
| `online_exam_prisma_query_duration_seconds` | Prisma 查询耗时直方图，不包含 SQL 文本和参数 |
| `online_exam_hydro_operations_total` | Hydro 登录、提交、机器人阻断结果 |
| `online_exam_hydro_operation_duration_seconds` | Hydro 外部调用耗时 |
| `online_exam_export_queue_depth` | 导出待处理队列深度 |
| `online_exam_export_jobs_total` | 导出领取、成功、重试、失败和租约恢复 |
| `online_exam_student_attempt_operations_total` | 自动保存、提交、强制结算和并发冲突 |

## 3. 建议告警规则

以下表达式以 5 分钟窗口为默认值，首次上线先观察一周基线再收紧。

```promql
# 核心 API 5xx 比例超过 2%，持续 10 分钟
sum(rate(online_exam_http_requests_total{status=~"5.."}[5m]))
/
sum(rate(online_exam_http_requests_total[5m])) > 0.02

# HTTP P95 超过 1.5 秒，持续 10 分钟
histogram_quantile(0.95, sum by (le) (rate(online_exam_http_request_duration_seconds_bucket[5m]))) > 1.5

# Prisma P95 超过 300ms，持续 10 分钟
histogram_quantile(0.95, sum by (le) (rate(online_exam_prisma_query_duration_seconds_bucket[5m]))) > 0.3

# 导出队列积压超过 20，持续 15 分钟
online_exam_export_queue_depth{status="pending"} > 20

# Hydro 机器人阻断或失败出现
increase(online_exam_hydro_operations_total{outcome=~"blocked|failed|rejected"}[10m]) > 0

# 自动保存失败比例超过 1%
sum(rate(online_exam_student_attempt_operations_total{operation="autosave",outcome="failed"}[5m]))
/
sum(rate(online_exam_student_attempt_operations_total{operation="autosave"}[5m])) > 0.01
```

告警通知必须包含环境、路由或任务类型、当前值、阈值、开始时间、日志检索用 request/correlation id，以及对应处置手册链接。

## 4. 数据库基线

运行 `pnpm db:baseline` 获取：连接/等待状态、最长事务、`pg_stat_statements` 慢查询 Top 20（扩展存在时）以及题库、考试记录、导出队列三个只读 `EXPLAIN (ANALYZE, BUFFERS)` 计划。

HTTP P50/P95/P99 从 Prometheus 直方图计算；数据库耗时和 SQL/request 可由 Prisma 指标与 HTTP 请求速率组合。新增索引前必须保存执行计划，确认过滤条件、扫描行数和排序代价，不凭感觉加索引。分页统一使用唯一键作为第二排序键，例如 `created_at, id`。

## 5. 审计与事件

- 业务审计继续写入独立 `audit_logs`，普通应用日志不能替代审计记录。
- 审计数据不得包含密钥、密码、Token 或完整学生答案。
- 涉及通知、第三方回写或跨模块异步副作用的新事务，采用同一数据库事务写入 Outbox 事件；独立 worker 按事件 ID 幂等投递并记录重试。当前不因这一方向引入新的消息中间件。
- 导出任务以 PostgreSQL 租约、心跳和原子领取实现可恢复处理；多实例规模超过数据库队列承载能力时再替换为 BullMQ/Redis。

## 6. 安全与依赖

- `pnpm security:audit` 对生产依赖执行高危漏洞门禁，使用官方 npm 审计端点。
- `pnpm licenses:check` 对生产依赖执行许可证白名单；未知许可证必须逐包复核并显式登记。
- 生产环境默认关闭 Swagger，CORS 必须为明确来源，JWT/附件签名/凭据加密密钥必须满足启动校验。
- 上传同时校验权限、大小、扩展名、MIME 与文件魔数；SVG 和可执行脚本不允许作为题目附件上传。
